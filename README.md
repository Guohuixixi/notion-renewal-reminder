# Notion 商业版账号到期提醒

这是一个基于 **Notion 数据库 + GitHub Actions + QQ 邮箱 SMTP** 的自动到期提醒工具。

## 功能说明

该项目用于管理 Notion 商业版账号的服务到期提醒。  
用户只需要在 Notion 表格中维护账号信息、服务开始日期、服务到期日期、是否续费等字段，GitHub Actions 会按照设定时间自动运行脚本，检查即将到期的账号，并发送邮件提醒。

## 工作流程

1. 在 Notion 数据库中维护商业版账号记录
2. GitHub Actions 按计划定时执行 Python 脚本
3. 脚本调用 Notion API 查询符合条件的记录
4. 找出即将到期、未续费、未发送过提醒的账号
5. 通过 QQ 邮箱 SMTP 发送提醒邮件
6. 邮件发送成功后，自动回写 Notion，将「提醒已发送」勾选，避免重复提醒

## 提醒规则

默认规则为：

- 服务到期日为目标提醒日期
- `是否已续费` 未勾选
- `提醒已发送` 未勾选

满足以上条件时，系统会发送提醒邮件到指定邮箱。

## Notion 表字段

Notion 数据库主要字段包括：

| 字段名 | 说明 |
|---|---|
| 用户账号 | 需要提醒到期的商业版账号 |
| 商业版试用账号 | 对应的商业版试用账号 |
| 商业版试用月额度 | 商业版试用账号的月额度 |
| 买家邮箱 | 买家或客户邮箱 |
| 联系邮箱 | 联系客户使用的邮箱 |
| 套餐类型 | 试用、正式、续费等 |
| 月数 | 本次服务时长 |
| 服务开始日 | 服务开始日期 |
| 服务到期日 | 服务结束日期 |
| 是否已续费 | 已续费则不会发送提醒 |
| 提醒已发送 | 邮件发送成功后自动勾选 |
| 通知邮箱 | 单条记录指定收件邮箱，留空则使用默认收件邮箱 |
| 备注 | 其他补充信息 |

## 邮件内容

提醒邮件会包含以下信息：

- 用户账号
- 商业版试用账号
- 商业版试用月额度
- 套餐类型
- 服务月数
- 服务开始日
- 服务到期日
- 买家邮箱
- 联系邮箱
- 备注
- Notion 记录链接

## 环境变量

GitHub Actions 需要配置以下 Secrets：

| Secret 名称 | 说明 |
|---|---|
| `NOTION_TOKEN` | Notion Integration Token |
| `NOTION_DATABASE_ID` | Notion 数据库 ID |
| `SMTP_HOST` | SMTP 服务器，例如 `smtp.qq.com` |
| `SMTP_PORT` | SMTP 端口，例如 `465` |
| `SMTP_USER` | 发件邮箱账号 |
| `SMTP_PASSWORD` | 邮箱 SMTP 授权码 |
| `MAIL_FROM` | 发件邮箱 |
| `MAIL_TO` | 默认收件邮箱 |

## 防重复提醒

脚本发送邮件成功后，会自动将该记录的 `提醒已发送` 字段勾选。  
后续即使 GitHub Actions 再次运行，也不会重复发送同一条提醒。

如果需要重新发送某条提醒，可以手动取消该记录的 `提醒已发送` 勾选。
